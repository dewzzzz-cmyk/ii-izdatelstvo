# Новые жанры, выпадающая «Эпоха», топ-15 авторских стилей — дизайн

## Цель

Три независимых, но тематически связанных дополнения к «Литсовету»:

1. Четыре новых жанра в `GENRES`, которых не хватало: ироничный детектив, ироничное фэнтези (в духе серии «МИФ» Роберта Асприна), сказка, юмористическая проза — каждый с реальными генерационными инструкциями (biты/тон/поправка судьи), а не просто ярлыком.
2. Поле «Эпоха» в Концепции становится выпадающим списком частых периодов + свободный текст для остального (мирроринг механизма, уже работающего у «Жанра»).
3. Стадия «Голос» получает библиотеку из 15 готовых авторских стилей (классика + жанровые авторы) — по клику добавляют имя автора в «Ориентиры стиля» и набор конкретных do/don't-правил в «Правила автора», переиспользуя уже существующий механизм (`addRule`, паттерн карточек как у разбора манеры письма).

Все три пункта — чисто аддитивные: новый код не меняет поведение существующих жанров/полей/правил, только добавляет данные и один новый UI-блок.

### 1.2 Важно про порядок веток в `genreBeatsNote`

`литсовет/src/genres.js:38-62` уже содержит явное правило: специфичные жанры проверяются раньше общих, потому что их названия — подстроки общих (пример из существующего кода: «дамский детектив» содержит «детектив»). Новая строка `'ироничный детектив'` **тоже** содержит подстроку `'детектив'` — значит её ветка обязана стоять раньше общей ветки `g.includes('детектив') || g.includes('триллер') || g.includes('мистик')` (строка 55-57), иначе общая ветка перехватит её первой и жанр не получит свою инструкцию.

Остальные три новых жанра (`ироничное фэнтези`, `сказка`, `юмористическая проза`) ни с чем существующим не пересекаются подстроками — но для единообразия и на случай будущих жанров все 4 новые ветки добавляются одним блоком **в самое начало** каждой из трёх функций (`genreBeatsNote`, `genreToneNote`, `genreJudgeNote`), сразу после вычисления `const g = (genre||'').toLowerCase();`, перед существующей веткой «альтернативная история».

### 1.3 Содержимое веток

**`genreBeatsNote`** (структурные биты для Книжного архитектора):

- **Ироничный детектив**: держит честную детективную основу (расстановка улик, минимум одна красная сельдь, развязка логично следует из улик — как в обычном детективе/триллере), поверх неё — комический контракт: рассказчик/сыщик имеет право на остроты и ироничные отступления даже в сценах с трупом, второстепенные персонажи могут иметь комические лейтмотивы («коронные фразы»), допрос/сбор улик — повод для комических сцен-перепалок, а не только нагнетания. Тяжесть преступления не обнуляется, но реакция персонажей на неё остаётся лёгкой.
- **Ироничное фэнтези**: держит фэнтезийный каркас (магическая система, квестовый сюжет), но последовательно его подрывает — магия как источник бюрократических/бытовых проблем (профсоюзы драконов, бумажная волокита в гильдии магов, похмелье после телепортации), протагонист может комментировать штампы жанра изнутри. Кульминация не обязана «серьёзнеть» — можно закрыть даже финальную битву панчлайном, не срывая ставки сюжета.
- **Сказка**: зачин и концовка-формула («жили-были…», «и стали они жить-поживать, да добра наживать» — вариации допустимы, сама формула нет), правило троекратности (три попытки/три брата/три испытания), архетипные роли (герой, вредитель, даритель/помощник, задание-испытание), явная мораль, прорастающая из развязки, а не проговорённая отдельной репликой в лоб. Магия — данность, не требующая обоснования.
- **Юмористическая проза**: структура комической сцены — завязка нелепости → нарастающая эскалация → панчлайн, а не одноразовая шутка без развития. Сквозные гэги должны возвращаться и «выстреливать» позже. Комизм строится на несоответствии ожиданий и реальности, а не на прямом заявлении рассказчика «это было смешно».

**`genreToneNote`** (тон/тропы уровня сцены для Прозаика):

- **Ироничный детектив**: рассказчику/сыщику разрешена ирония и остроумие даже в напряжённых по фабуле сценах — это конвенция жанра, не потеря достоверности. Комический тайминг реплик (подводка → сбивка) важнее «реалистичности» темпа речи в допросе.
- **Ироничное фэнтези**: можно снижать пафос «эпического» момента шуткой или бытовой деталью. Магию уместно описывать механически/бюрократически ради комического эффекта.
- **Сказка**: простой, ритмичный, устно-сказительный регистр; повтор («шёл-шёл», «жил-был») — стилистический приём, не бедность языка. Психологическая глубина персонажей не требуется — архетипность уместна.
- **Юмористическая проза**: гипербола, абсурдная логика и саморазоблачающий рассказчик — уместные приёмы. Недопонимание между персонажами — законный двигатель сцены.

**`genreJudgeNote`** (поправка для Оценщика/Стражей — зеркало тона, но для контролёров качества):

- **Ироничный детектив**: лёгкий/иронический тон при описании убийства/улик — не провал ставки и не тональный сбой, жанровый контракт. Не требовать мрачной серьёзности процедурала.
- **Ироничное фэнтези**: снижение пафоса эпического момента шуткой — не тональный сбой и не потеря ставок. Не штрафовать бюрократическое/бытовое описание магии как «обесценивание» мира.
- **Сказка**: плоские архетипные персонажи и повторяющиеся формульные фразы — не слабость характеристики и не повтор-ошибка, а жанровая конвенция.
- **Юмористическая проза**: гипербола и абсурдная логика — не «нереалистичность» и не логическая дыра. Не штрафовать комическое недопонимание персонажей как слабый конфликт.

(Точные финальные формулировки — на усмотрение реализующего: план должен включать полный текст каждой ветки, в стиле уже существующих — конкретно, императивно, без общих слов.)

### 3.2 Опциональный хелпер

```js
export function styleMatchesGenre(style, genre){
  return !!genre && style.genres.includes(genre);
}
```
(используется UI-слоем для сортировки/бейджа — см. §3.4.)

### 3.3 Содержимое `rules[]` — 3-4 строки на стиль

Формат — тот же, что у существующего `STARTER_RULES` (`литсовет/src/ui/stages.js:507-513`): конкретные, императивные do/don't-инструкции, без общих слов вроде «пиши хорошо». План должен зафиксировать полный текст для всех 15 стилей; ориентир по духу:

- *Чехов*: «Не называй эмоцию персонажа впрямую — покажи через деталь, жест, паузу.», «Реплики диалога обрывай на характерном слове, не досказывай до конца.», «Ирония — через несоответствие тона и события, не через прямую насмешку рассказчика.»

- *Пратчетт*: «Уместны сноски-отступления с параллельным ироничным комментарием к основному действию.», «Магию/систему мира описывай с бюрократической серьёзностью — это и есть шутка.»
- *Хемингуэй*: «Убирай прилагательные, если смысл ясен без них — эффект «айсберга» держится на недосказанности, не на украшении.», «Эмоция передаётся через действие и диалог, не через авторскую характеристику состояния.»

### 3.4 UI в «Голосе» (`литсовет/src/ui/stages.js`, функция `renderVoice`)

Новый блок «Готовые стили» вставляется в `renderVoice`'s разметку **перед** `renderRefsEditor(s)` (строка 390) — потому что применение стиля пишет и в «Ориентиры», и в «Правила» ниже.

Рендер — карточки, тем же визуальным паттерном, что `renderMannerCards` (строки 456-474):

```js
function renderStylePresets(s){
  const genre = s.project.genre;
  const applied = new Set(s.style.refs||[]);
  const sorted = [...AUTHOR_STYLES].sort((a,b)=>
    (styleMatchesGenre(b,genre)?1:0) - (styleMatchesGenre(a,genre)?1:0));
  return `<div class="field" style="margin-top:22px;border-top:1px solid var(--border);padding-top:16px">
    <label>Готовые стили <span class="hint">(классика и жанровые авторы — добавляют имя в «Ориентиры» и набор правил в «Правила автора»)</span></label>
    <div style="display:grid;grid-template-columns:repeat(auto-fill,minmax(220px,1fr));gap:8px;margin-top:8px">
      ${sorted.map(st=>{
        const isApplied = applied.has(st.name);
        const fits = styleMatchesGenre(st, genre);
        return `<div class="card" style="padding:8px 10px">
          <div style="font-weight:600;font-size:13px">${esc(st.name)}${fits?' <span class="muted" style="font-weight:400;font-size:11px">· подходит жанру</span>':''}</div>
          <div class="muted" style="font-size:12px;margin:4px 0 8px">${esc(st.blurb)}</div>
          <button class="btn style-add" data-id="${st.id}" ${isApplied?'disabled':''} style="font-size:11px;padding:3px 9px">${isApplied?'✓ Добавлено':'+ Добавить'}</button>
        </div>`;
      }).join('')}
    </div>
  </div>`;
}
```

Обработчик клика (мирроринг `manner-add`, строки 465-473 — `addRule` уже дедуплицирует через `sameNote()`, `литсовет/src/state.js:181-186`, так что повторное применение пересекающихся стилей безопасно):

```js
document.querySelectorAll('.style-add').forEach(btn=>{
  btn.onclick = ()=>{
    const st = AUTHOR_STYLES.find(s=>s.id===btn.dataset.id); if(!st) return;
    const s = getState();
    s.style.refs = s.style.refs || [];
    if(!s.style.refs.includes(st.name)) s.style.refs.push(st.name);
    st.rules.forEach(r=>addRule(s, r));
    save();
  };
});
```

Кнопка становится неактивной («✓ Добавлено») после применения — определяется по наличию имени автора в `s.style.refs`, без отдельного флага состояния. Повторное нажатие невозможно (кнопка `disabled`), что и требуется («не эксклюзивно, но не бесконечно повторяемо для одного стиля»).
