# Литсовет — вкладка «Мир»: карточки по категориям — Design Spec

**Дата:** 2026-07-05
**Статус:** утверждён к планированию
**Тип:** переработка существующей стадии «Мир» (не новая подсистема)

---

## 1. Проблема

Стадия «Мир» (`src/world.js`, `src/ui/world.js`) сейчас даёт только один путь: кнопка
«✨ Предложить мир» одним вызовом ИИ выдаёт 8-15 фактов сразу по всем категориям
(география/история/фракции/культура + магия-технология или система по жанру).
Из этого вытекают четыре ограничения для «настоящего писателя»:

1. **Нет точечной генерации.** Если автор доволен географией, но хочет добавить только
   фракции — приходится либо звать «Предложить мир» заново (все категории, лишние
   кандидаты), либо писать вручную.
2. **Канон — только для чтения.** Секция «Уже в каноне» (`renderCanon`) выводит текст
   без единой кнопки. Поправить формулировку или удалить факт, который передумали
   использовать, с вкладки «Мир» нельзя — хотя *в другом месте приложения* (вкладка
   «Написание» → панель «Память», `ui/memory.js:261-280`) правка/удаление/AI-расширение
   произвольного факта Библии уже реализовано. Дублировать эту возможность именно на
   «Мире» никто не удосужился.
3. **Подсказки — не по адресу.** Три общих поля справа («Идея мира», «Ограничение
   магии/технологии», «Антагонистичная фракция») светятся всегда, даже если открыта
   генерация только географии — семантически два из них относятся к конкретным
   категориям, а показаны как настройки всего мира.
4. **Нет наглядности проработанности.** Список фактов плоский — не видно с одного
   взгляда, какие категории уже насыщены, а какие пустуют.

## 2. Цели и не-цели

### Цели
- Генерация **по одной категории** — не только «весь мир одним запросом».
- Правка и удаление уже сохранённых в канон world-фактов **прямо на «Мире»**.
- Возможность добавить факт руками, без ИИ.
- Подсказка привязана к своей категории, а не свалена в общий блок.
- Перегенерировать (переформулировать с нуля) один конкретный факт, если формулировка
  не устроила.
- Наглядно видно, какая категория уже проработана, а какая пуста.
- «Предложить весь мир» как было — остаётся, для тех, кто хочет один клик.

### Не-цели (YAGNI)
- Версионирование/история правок world-фактов (у Библии сейчас нет истории версий
  нигде, в т.ч. в панели «Память» — не заводим её только для «Мира»).
- Отдельная страница/модалка для правки факта — переиспользуем существующий паттерн
  `prompt()`-диалогов из `ui/memory.js`, не проектируем новый инлайн-редактор.
- Изменение автоматической подсказки Архитектора о недостающих фактах
  (`suggestMissingWorldFacts`, стадия «Структура») — не трогаем, она уже работает
  независимо через `state.bible`.
- Изменение карты мира и кнопки «Дальше» — не в скоупе.

## 3. Раскладка: карточка на категорию

`renderWorld()` вместо одного блока «Предложить» + плоских списков рендерит по одной
карточке на каждую категорию из уже существующей `categoriesFor(project.genre)`
(`world.js` — набор не меняется, всё ещё жанрово-осознанный: магия/технология для
world-жанров, система для ЛитРПГ). Порядок карточек = порядок `categoriesFor()`.

Каждая карточка содержит:
- Заголовок: название категории + счётчик («География · 3 факта» / «Фракции · пусто»)
  — это и есть требуемая «наглядность проработанности» (§2), отдельный компонент не
  нужен, счётчик уже есть в структуре карточки.
- Поле подсказки для этой категории (необязательное, однострочное, placeholder на
  основе уже существующего `CATEGORY_HINTS[cat]` — не новый текст, а то же описание,
  что сейчас уходит в системный промпт агента).
- Кнопка «✨ Предложить» — генерация **только для этой категории** (см. §5).
- Список фактов уже в каноне этой категории, каждый с действиями (см. §4).
- Кандидаты этой категории, ещё не одобренные (тот же чекбокс+редактирование+«Сохранить
  в канон», что и сейчас в `renderCandidates`, просто область видимости — одна
  категория, а не «Кандидаты» общим блоком сверху).
- Ссылка «+ добавить вручную» (см. §4.3).

Наверху экрана, над карточками, остаётся кнопка «✨ Предложить весь мир» — не убираем
путь в один клик, см. §5.2. Карта мира (`renderMapBlock`) и кнопка «Дальше» — без
изменений, в конце страницы, как сейчас.

Правая панель (`els.right`) сжимается до одного поля — «Идея мира» (общий контекст
всего сеттинга, не привязан к категории). Поля «Ограничение…» и «Антагонист…» из неё
убираются — их роль теперь играют подсказки категорий «Магия/технология» и «Фракции»
соответственно (§3, №2).

## 4. Действия над фактом — переиспользуем `ui/memory.js`, не изобретаем заново

`ui/memory.js:261-278` уже реализует для произвольной записи Библии: ✎ правка через
`prompt()`, ✕ удаление, ✨ расширение через ИИ (`archivist`-подобный промпт, дописывает
факт деталями). План — вынести общую часть этого обработчика (сейчас завязана на
индекс `s.bible[i]` и `data-bi`) в переиспользуемую функцию, вызываемую из обоих
файлов, а не копировать код листингом. Точное имя/расположение хелпера — решение
implementation-плана (кандидат: `bindBibleFactActions()` в `src/bible.js`, рядом с
`rebuildBibleVecs`, — уже общее место для операций над `state.bible`).

### 4.2 Новое: 🔄 «Другой вариант»
В отличие от ✨-расширения (`ui/memory.js`, дописывает факт деталями, не меняя суть) —
просит ИИ **переформулировать факт заново**, для случая «эта формулировка не
нравится». Новая функция `rerollWorldFact(state, fact)` в `src/world.js`: системный
промпт «Ты — соавтор по мироустройству. Тебе не нравится формулировка факта — предложи
другую версию ТОГО ЖЕ факта (та же суть, другие слова), без изменения смысла».
Результат заменяет `fact.text` напрямую (по аналогии с `regenerateScene()` в
`architect-book.js` — там перегенерация сцены тоже заменяет поля на месте, без
режима «принять/отклонить»); `rebuildBibleVecs()` после замены.

### 4.3 Добавить вручную
Ссылка «+ добавить вручную» внизу каждой карточки открывает те же два `prompt()`,
что уже использует `bibleAdd` в `ui/memory.js` («Ключи факта» → «Сам факт»), но
category проставляется автоматически (категория этой карточки) — автору не нужно
её называть самому.

## 5. Генерация: по категории и «весь мир»

### 5.1 По одной категории
`suggestWorldFacts(state, hints)` → `suggestWorldFacts(state, category, hint)`.
`worldSuggestMessages()` строит промпт для **одной** категории вместо списка всех —
проще и точнее промпт (агент не размазывается на 4-5 категорий сразу), `hint` —
текст из поля подсказки этой карточки (заменяет три старых поля `ideaSeed`/
`limitation`/`antagonistFaction` одним generic `hint` на категорию + отдельным
глобальным `ideaSeed`, см. §3). Возвращает 3-6 фактов (меньше, чем 8-15 «на весь
мир» — одна категория физически не требует полутора десятков фактов).

### 5.2 «Предложить весь мир» — один клик, тот же путь
Кнопка наверху не делает отдельный «большой» вызов ИИ на все категории сразу (как
сейчас) — вместо этого вызывает `suggestWorldFacts` для каждой категории
`categoriesFor(genre)` **последовательно, одна за другой** (не параллельно — все
категории пишут в общий `_candidates`, и одновременные `await`-вызовы гонялись бы
за одним и тем же состоянием), используя глобальную «Идею мира» как общий `hint`,
если поле категории пустое. Один путь генерации вместо двух (точечного и
«массового») — меньше кода для поддержки, каждый вызов честно сфокусирован на одной
категории. Компромисс: вместо одного вызова ИИ на весь мир — 4-5 отдельных вызовов
(тот же текстовый LLM, что и для прозы — без отдельных денег, только больше токенов
накладных расходов на системные промпты и общее время ожидания). Кнопка показывает
общий прогресс («Мир»: 2 из 4 категорий готовы…) вместо одного спиннера.

## 7. Проверка (live-тест)

1. Открыть «Мир» на жанре с world-фактором (напр. «Фэнтези») → видно по одной
   карточке на «География/История/Фракции/Культура/Магия-технология», у каждой свой
   счётчик и поле подсказки.
2. Заполнить подсказку только у «Фракции», нажать «✨ Предложить» именно там (не
   «весь мир») → реальный вызов ИИ, приходят кандидаты только этой категории.
3. Одобрить кандидата → попадает в «уже в каноне» этой карточки со счётчиком +1, виден
   и в панели «Память» на «Написании» (тот же `state.bible`).
4. У факта в каноне нажать ✎ → `prompt()` с текущим текстом → изменить → сохранился;
   ✕ → удалился, счётчик карточки -1.
5. Нажать 🔄 «Другой вариант» у факта → реальный вызов ИИ → текст заменился на другую
   формулировку той же сути (не то же самое дословно).
6. «+ добавить вручную» на карточке «Культура» → два `prompt()` → факт появился с
   `category:'культура'` без единого вызова ИИ.
7. Наверху «✨ Предложить весь мир» на пустом мире → последовательно заполняются все
   карточки (или показывается прогресс), в сумме факты по всем категориям жанра.
8. Перейти на «Структуру», собрать скелет → `suggestMissingWorldFacts` продолжает
   работать как раньше (не в скоупе правок, но не должен был сломаться).
