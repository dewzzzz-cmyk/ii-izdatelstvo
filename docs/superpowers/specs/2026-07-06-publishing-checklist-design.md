# Литсовет — раздел «Публикация»: готовый экспорт + чек-лист под ручную загрузку — Design Spec

**Дата:** 2026-07-06
**Статус:** утверждён к планированию
**Тип:** новая стадия в существующем пайплайне «Литсовет»

---

## 1. Проблема

Автор, дописав книгу в Литсовете, вручную готовит материалы для публикации на
внешних площадках (Author.Today, ЛитРес Самиздат, Проза.ру/Стихи.ру) — копирует
текст по главам, следит за лимитами знаков, помнит про формат обложки и
метаданные — без единого места, которое бы напоминало, что именно и в каком
виде нужно каждой площадке.

**Проверено перед проектированием (важно, ограничивает весь скоуп):** ни у
одной из трёх площадок нет доступного автору API для автоматической публикации:

- **Author.Today** — API есть (`api.author.today`), но **только для чтения**
  (каталог, профиль, статистика, лайки — то, что использует их мобильное
  приложение). Методов создания/загрузки/публикации произведений нет вообще.
- **ЛитРес Самиздат** — API для публикации существует и реально умеет нужное
  (загрузка FB2/DOCX/PDF, метаданные, обложка, цена), но ключи доступа
  выдаются **только после заключения договора** с ЛитРес через их менеджеров —
  это бизнес-процесс, не самостоятельная регистрация разработчика. Вне скоупа
  этой фичи.
- **Проза.ру / Стихи.ру** — публичного API нет вообще, только веб-форма
  (`proza.ru/cgi-bin/publish.pl`) для ручной публикации.

Поэтому эта фича — **не автоматическая публикация**, а подготовка материалов
в точном виде, который ждёт ручная форма/загрузчик каждой площадки, плюс
чек-лист их конкретных требований.

## 2. Цели и не-цели

### Цели
- Раздел «Публикация» — новая стадия, по одной секции на площадку.
- **Author.Today / Проза.ру** (текст вставляется в веб-форму по главам/целиком,
  файла нет): показать книгу по главам с кнопкой «📋 Копировать» на каждую —
  переиспользует уже реализованный в «Редактуре» паттерн копирования в буфер.
  Явно предупреждать о лимите ~100 000 знаков на «часть» (Author.Today) и
  показывать счётчик знаков главы.
- **ЛитРес Самиздат** (файл целиком — DOCX/FB2/PDF): новый **настоящий FB2**
  экспорт — простой, честно валидный XML-формат, в отличие от нынешнего
  HTML-в-.doc трюка, который рискует не пройти строгую валидацию их
  загрузчика. Чек-лист: не дублировать метаданные в самом файле — их отдельно
  заполняют в мастере публикации на сайте (иначе конфликт при конвертации).
- Чек-лист на каждую площадку — конкретные, проверенные фактами требования
  (мин/макс объём, формат обложки, где заполнять метаданные), не общие фразы.

### Не-цели (YAGNI)
- Реальная автоматическая публикация через API — технически недоступна для
  всех трёх площадок (см. §1), не проектируем «на будущее».
- Партнёрская интеграция с ЛитРес API — юридический/бизнес-шаг, не код.
- Поддержка других площадок (Wattpad, Ridero и т.п.) — не запрашивались,
  добавляются по тому же паттерну позже, если понадобится.
- OAuth/логин через площадки — раз нет API, авторизовываться не для чего.

## 3. Раскладка: аккордеон по площадкам

Новая стадия `publish` (`{ id:'publish', label:'Публикация' }`), последняя в
`STAGES` (после «Редактура» — публикация имеет смысл только для готовой,
вычитанной книги). Видна всегда (как «Иллюстрации»/«Мир» — без скрытого
тумблера, YAGNI после разбора с «Миром» в этой же сессии).

Одна карточка-секция на площадку (переиспользует `.world-cat-card`-подобный
контейнер для визуальной консистентности с «Миром»):

- **Author.Today**: чек-лист (лимит ~100k знаков/часть, порог 15k для
  «Новинок», где на сайте добавлять части) + список глав, у каждой счётчик
  знаков и кнопка «📋 Копировать текст главы».
- **Проза.ру / Стихи.ру**: тот же паттерн (список произведений для копирования
  + краткий чек-лист их формы публикации).
- **ЛитРес Самиздат**: чек-лист (мин. 4000 знаков, макс. 70 МБ, формат
  обложки — PNG/JPG RGB, метаданные — в мастере на сайте, не в файле) + кнопка
  «⬇ Скачать FB2».

## 4. Новый экспорт: честный FB2

FB2 — простой, документированный XML-формат (`<FictionBook>`), в отличие от
нынешнего `.doc` (HTML с расширением .doc, см. `exportDocx()`). Новая функция
`exportFb2(state)` в `литсовет/src/export.js`, переиспользует уже существующий
`buildBook(state)` (даёт `{title, chapters:[{title, scenes:[{id,title,text}]}]}`
— тот же хелпер, что и `.md`/`.docx`/EPUB) и `typo()` для типографики.

**Важно (найдено спек-ревью):** `buildBook` сейчас объявлена без `export` —
приватна для `export.js`, её же вызывают `exportMd`/`exportDocx`/`exportEpub`
внутри самого файла. Так как `ui/publish.js` (см. §6) — отдельный модуль,
которому тоже нужен доступ к `buildBook` (для копирования по главам, §5) — при
реализации первым шагом добавить `export` к `buildBook` (и, если понадобится
показывать иллюстрации сцен в предпросмотре копирования — к `illustrationForScene`
тоже). Не изобретать копию функции — расшарить существующую.

Минимальная структура FB2 (без метаданных автора/жанра/аннотации в файле —
по чек-листу ЛитРес это сознательно оставляется мастеру публикации на сайте,
чтобы не конфликтовать при конвертации):

```xml
<?xml version="1.0" encoding="UTF-8"?>
<FictionBook xmlns="http://www.gribuser.ru/xml/fictionbook/2.0">
  <description>
    <title-info><book-title>{Название}</book-title></title-info>
  </description>
  <body>
    <section><title><p>{Глава 1}</p></title>
      <p>{текст сцены 1}</p>
      <p>{текст сцены 2}</p>
    </section>
    ...
  </body>
</FictionBook>
```

Обложка — намеренно НЕ встраивается в FB2 (в отличие от EPUB/`.doc`): чек-лист
ЛитРес прямо говорит грузить обложку отдельно через их мастер, встраивание
в файл создаёт дублирование источников правды. Кнопка «⬇ Скачать FB2» рядом
показывает подсказку «обложку и метаданные добавите на сайте — не встроены в файл специально».

## 5. Копирование по главам (Author.Today / Проза.ру)

Переиспользует ровно тот же паттерн `copyNoteBtn`/`bindCopyNotes`, что уже
реализован в `литсовет/src/ui/stages.js` для находок Бета-ридера/Критика —
кнопка кладёт текст в буфер, на 1200мс меняется на «✓ Скопировано». Каждая
глава книги (`buildBook(state).chapters`, сцены внутри склеены как в `.md`
экспорте — через `typo()` и разделитель `***`, БЕЗ пробелов между звёздочками
— именно так, как в `exportMd()`: `.join('\n\n***\n\n')`; не путать с
`* * *`-с-пробелами из EPUB CSS/`.docx`, это другая конвенция другого
экспортёра) получает такую кнопку + счётчик знаков рядом с заголовком,
подсвечивается красным если > 100 000 (предупреждение Author.Today про лимит
на часть).

## 6. Файловый план

| Файл | Действие | Назначение |
|---|---|---|
| `src/export.js` | изменить | новая `exportFb2(state)` — честный FB2-экспорт, переиспользует `buildBook`/`typo` |
| `src/ui/publish.js` | создать | рендер стадии «Публикация»: аккордеон по 3 площадкам, копирование по главам, кнопка FB2 |
| `src/ui/app.js` | изменить | `STAGES` — новая запись `publish` (последняя); `renderRail`/`renderStage` — ветка для стадии |
| `src/ui/stages.js` | изменить | `STAGE_LABELS`/`stageDoneFor` — синхронизировать вторую (независимую) карту стадий панели «Этапы производства» — тот же паттерн, что и при добавлении «Иллюстраций»/«Мира» |
| `src/styles.css` | изменить | стиль аккордеон-секций площадок (переиспользует `.world-cat-card`, новые правила по минимуму) |

## 7. Проверка (live-тест)

1. Открыть «Публикация» на книге с ≥2 главами → видны 3 секции-площадки.
2. Author.Today: у каждой главы свой счётчик знаков; глава >100k знаков —
   счётчик подсвечен; клик «📋 Копировать» → текст в буфере (реальная
   проверка через `navigator.clipboard`/подмену).
3. ЛитРес: клик «⬇ Скачать FB2» → скачался валидный XML (проверить
   `DOMParser` не бросает ошибку на разбор), название файла = название книги,
   в файле нет полей автора/жанра/цены/обложки.
4. Проза.ру: аналогично Author.Today — копирование работает.
5. Панель «Этапы производства» (правая панель «Написания»/«Редактуры»)
   показывает «Публикация» в списке стадий с верным статусом done/not-done.
