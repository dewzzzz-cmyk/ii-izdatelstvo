# ИИ-Издательство v2.0

Локальный редактор для написания книг с помощью LLM-агентов.  
Работает полностью offline (кроме API-вызовов к модели). Без npm, без сборки.

## Быстрый старт

### Требования
- **Node.js 18+** — https://nodejs.org

### Windows
```
start-windows.bat
```

### Linux / macOS
```bash
chmod +x start.sh
./start.sh
```

### Вручную
```bash
node server.js
```

Откройте браузер: **http://localhost:8787**

## Первый запуск
1. Откройте ⚙ **Настройки**
2. Выберите провайдера (DeepSeek, OpenAI, OpenRouter…)  
3. Вставьте API-ключ
4. Нажмите **Сохранить**
5. Выберите шаблон в **🗂 Шаблоны** — и нажмите **▶ Запустить**

**DeepSeek** — самый дешёвый вариант (~$0.01 за рассказ).  
Получить ключ: https://platform.deepseek.com/api_keys

## Структура файлов
```
server.js       — прокси-сервер (Node.js, port 8787)
app.js          — весь frontend (~5000 строк vanilla JS)
index.html      — HTML-оболочка
styles.css      — стили
backups/        — создаётся автоматически: JSON + .md книги
```

## Переменные окружения
| Переменная | Описание |
|---|---|
| `PORT` | Порт сервера (по умолчанию 8787) |
| `PROXY_TOKEN` | Токен доступа (если разворачиваете публично) |

## Где хранятся книги

Книги сохраняются автоматически в **двух местах**:

### На диске (надёжно)
```
backups/
├── НазваниеКниги_2026-05-30.md    ← читаемый текст (Markdown)
└── backup_2026-05-30T19-33.json   ← полный снимок для восстановления
```
Папка `backups/` создаётся рядом с `server.js` при первом прогоне.  
`.md` файл открывается в Word, Obsidian, любом редакторе.

### В браузере (временно)
- `localStorage` — текущий проект
- `IndexedDB` — автосохранение каждые 10 минут

> ⚠️ Данные в браузере могут исчезнуть при очистке. Всегда используйте **backups/**.

### Перенос книг на другой компьютер
**Способ 1 (все книги):** скопировать папку `backups/`  
**Способ 2 (через приложение):** Мои книги → «⬇ Скачать библиотеку» → на новом ПК «📥 Импорт»
